# 학습 인덱스 (Map of Content)

에이전트 간 공유되는 학습을 토픽별로 묶는다. 세션 시작 시 본인 learnings 파일을 읽은 뒤, 관련 토픽이 있으면 이 인덱스에서 다른 에이전트의 학습도 참조한다.

## 자동화-안정성
> PATH 의존, self-healing, LaunchAgent, 바이너리 탐색

| 날짜 | 에이전트 | 요약 | 파일 |
|------|---------|------|------|
| 2026-03-21 | 닉 퓨리(CEO) | LaunchAgent PATH 누락 → 바이너리 직접 탐색 필수 | [[ceo]] |

**교훈**: 모든 자동화 스크립트는 `PATH` 환경 변수에 의존하지 말고, 바이너리를 직접 탐색해야 한다. 이 원칙은 자비스(인프라), 스파이더맨(개발), 헐크(R&D) 모두에게 적용된다.

| 날짜 | 에이전트 | 요약 | 파일 |

| 2026-03-21 | 닉 퓨리(CEO) | 가즈아 체계 이식 (감찰관/learnings/산출물 기준) | [[ceo]] |
| 2026-03-21 | 블랙 위도우 | 컴플라이언스에 감찰관 역할 추가 | [[compliance]] |

**교훈**: done 게이트키핑은 전담 에이전트(비전)가 담당. 산출물 없이 done 처리되면 자동 거부. 이 원칙은 전체 에이전트에 적용.

| 2026-03-23 | 시스템 | 메타 이슈 생성만 하고 소멸 경로 누락 → GC 추가 | [[ceo]], [[inspector]] |

**교훈**: 자동 생성되는 메타 이슈([보고], [라우팅], [복구 위임])는 반드시 **소멸 조건**도 함께 설계. 원본 이슈가 해결되면 메타 이슈는 자동 cancelled. `check_paperclip_issues.py`의 가비지 컬렉션이 이를 강제한다. **원칙: 생성 경로를 만들면 소멸 경로도 만들어라.**

## 이슈-관리
> 상태 변경, 코멘트 처리, wake-up 규칙

| 날짜 | 에이전트 | 요약 | 파일 |
|------|---------|------|------|
| 2026-03-21 | 콜슨(HR) | done 이슈 코멘트 wake-up 시 재개입 불필요 (단순 정리면) | [[hr]] |

**교훈**: done 이슈에 코멘트가 달려도, 그것이 단순 완료 정리라면 추가 조치 불필요. 불필요한 재개입은 작업 효율을 떨어뜨린다.

| 2026-03-23 | 제갈량(CEO) | 네이버 업로드 실패 → 원인 파악 없이 무한 재시도 → 프로세스 누적 | [[ceo]] |

**교훈**: 에러 발생 시 **진단이 먼저**. 같은 명령을 반복 실행하는 것은 해결이 아니라 자원 낭비. 원인 분석 → 해결 가능하면 1회 재시도, 불가능하면 유지보수 요청 후 중단. **원칙: 진단 먼저, 재시도는 1회만.**

| 2026-03-23 | 시스템 | 이슈 cancelled인데 activeRun running → 좀비 run이 네이버 업로드 반복 | [[ceo]], [[inspector]] |

**교훈**: 이슈 상태 변경(cancelled/done) 전에 **activeRun 확인 필수**. run이 돌고 있으면 이슈만 닫아도 run은 계속 실행된다. **원칙: 이슈 닫기 전에 run 확인. 생성-소멸 경로에 run lifecycle도 포함.**
