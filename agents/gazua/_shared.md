## CLI 실행 실패 시 공통 워크플로우

**원칙: 진단 먼저, 재시도는 1회만, 같은 에러 반복 금지.**

스크립트 실행 중 에러 발생 시:
1. **원인 파악 먼저** — 에러 메시지, traceback, 로그를 읽고 원인 분석
2. **환경 문제(네트워크, 인증 만료 등)라면** → 1회 재시도 허용. 2회 연속 같은 에러면 중단
3. **코드 버그라면** → 재시도 금지. 즉시 유지보수 요청
4. `[유지보수]` 이슈 생성 (원인 분석 결과 포함):
   ```bash
   curl -s -X POST "$API/companies/$CID/issues" \
     -H 'Content-Type: application/json' \
     -d '{"title":"[유지보수] {스크립트명} 실행 에러","description":"## 에러\n{메시지}\n\n## 원인 분석\n{분석 결과}\n\n## 재현\n{명령}","status":"todo","labelIds":["150bfab9-225d-479c-9109-dc93710f6cac"]}'
   ```
5. 원래 이슈에 "[차단] 유지보수 요청" 코멘트
6. **해당 스크립트 재실행 중단** — 개수라발발타 스파이더맨이 수정할 때까지 대기

**절대 금지**: 같은 에러에 대한 무한 재시도. 원인 파악 없는 재시도. 코드 직접 수정.
